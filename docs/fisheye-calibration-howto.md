# 鱼眼相机标定操作手册 —— Calibration Workbench · 鱼眼标签页

> 器材：**12×9 金属棋盘板（内角点 11×8）、方格 4.5 cm、无反光**。
> 数据源：**ROS2 · 话题 `image_rect_compressed`**。
> 界面语言：**中文**（在「偏好设置 → 语言」切换）。下面的控件名均为软件里的中文标签。

---

## 0. 固有参数（每次都这么填，填一次基本不用动）

### 「源」面板（左上）
- 右上角开关切到 **`ros2`**。
- **「话题」** 下拉里选 **`image_rect_compressed`**（下拉只列 CompressedImage 话题，选项后括号是发布者数）。
  - 若下拉是空的（提示「未找到 CompressedImage 话题」）：用下面 **「手动」** 输入框手填完整话题名，再点 **「使用」**。
  - 若提示「rclpy 不可用 — 启动后端前请先 source ROS2 环境」：先 source 好 ROS2 环境再启动后端。
- 点 **「👁 实时预览」**，确认能看到相机画面。

### 「标定靶标」面板
| 控件 | 填什么 |
|---|---|
| 类型 | **棋盘格** |
| 列 × 行 | **11 × 8**（内角点，非方格数；12×9 方格 = 11×8 内角点） |
| 方格 | **0.045**（米） |
| 其余 | **默认** |

### 「投影模型」面板
- 保持默认 **等距 (equidistant)**。
- 下面几个勾选项（估计 k₁…k₄ / 光束法平差位姿 等）保持默认。

---

## 1. 选数据集文件夹

「数据集」面板 → 点 **「📁 选择文件夹」** → 选一个**空文件夹**当本次会话目录。
之后每次抓拍都存进这里。
点 **「清除」** 会弹确认框（「将此文件夹中的 N 张图像移入回收站？」），确认后整夹清空，可一次 **⌘Z** 撤销。

---

## 2. 各角度需要拍多少张（照这个清单加）

相机固定，**手举金属板移动**。每摆好一个姿态、停稳、画面里角点彩点画全了，
按 **空格** 或点 **「⌁ 抓拍帧」** 添加一张。按这个分配拍，共约 **30 张**：

| 组别 | 张数 | 怎么摆 / 范围 |
|---|---|---|
| 正对·分区 | 5 | 棋盘正对相机，依次放：中心、左上、右上、左下、右下 |
| 边缘 | 4 | 棋盘怼到画面 **上 / 下 / 左 / 右** 四条边（鱼眼边缘畸变最强，必拍） |
| 俯仰倾斜 tilt | 4 | 上沿前倒 / 后仰，**约 ±30°~45°**，各 2 张 |
| 偏摆倾斜 yaw | 4 | 左边前转 / 右边前转，**约 ±30°~45°** |
| 平面内旋转 roll | 2 | 棋盘正对相机，但当钟表转个角度 |
| 远近 | 3 | 拉近占满、推远变小、中等各 1 张 |
| 补漏 | 若干 | 看「采集」面板的 **8×5 覆盖网格**，哪个格子还黑，就把棋盘移到画面那个位置补上 |

整套图要让角点铺满 40 格网格（盯着 **「覆盖率」** 往 100% 凑）。

**棋盘占画面比例（建议）：**

| 占画面（面积） | 评价 | 说明 |
|---|---|---|
| 约 **1/3 ~ 1/2** | 推荐 | 放中心/分区时的常用大小，角点清晰、检测稳 |
| 怼边缘时 1/5 ~ 1/3 | 可以 | 边缘小一点没关系，但角点要清楚、整块在画面内 |
| > 2/3、快占满 | 太大 | 外圈白边/角点易被画面或鱼眼视场圈切掉 → 整帧作废 |
| < 1/6、很小一块 | 太小 | 角点太密、易糊，检测不准 |

> **省力技巧**：勾 **「自动采集」**（会自动开「实时检测」），调 **「自动速率」**（如 0.5s），
> 然后手举板缓慢移动 —— 软件只在棋盘进入**还没点亮的格子**时才自动抓拍，不会重复刷同一区域。

> **两种自动检测叠加（二选一，工具栏切换）**：
> - **「覆盖靶盘」**：上面说的极坐标飞镖盘，手持扫一遍把各环各扇区铺满即可。
> - **「文档引导」**：照本节这张清单**逐个位置 / 动作**走，圆框里高亮当前该把板放哪、什么姿态，
>   **每个动作自动拍两张**（第二张要你**稍微动一点点**，避免重复帧），拍够就语音/字幕提示
>   「下一个」。跟着提示走完整条序列，覆盖率自然到位。

### 三条铁律
1. **必须有倾斜的图**：全正对（0°）的帧会被求解器当病态帧自动丢掉，且焦距/距离分不开。别超过 ~60°（角点会糊）。
2. **停稳再拍**，避免运动模糊。
3. **棋盘整块在画面内**（含最外圈白边），被切掉的识别不出。

> 金属无反光板：基本不会因反光丢角点。只要光照均匀、别在板面打出强光亮斑。

## 3. 第一次运行标定

点左下 **「运行标定」**。
看右侧「结果」栏 **rms**，以及顶部状态条用了几帧 / 跳过几帧。

第一次往往不达标（rms 偏黄/偏红），按下面**两轮筛图**清掉坏帧。

---

## 4. 两轮筛图（关键，按顺序做）

### 第 1 轮：筛掉「数值明显发黄」的帧
- 看顶部 **帧条**：每张帧都标了它的重投影误差数字，带红绿灯颜色。
- **数值明显是黄色**的帧 = 误差偏高。
- 用 **← / →** 选中这种帧 → 点 **「⌧ 删除所选」**（或按 **Delete / Backspace**）删掉。误删可 **⌘Z** 撤销。
- 把明显发黄的都筛掉。

### 第 2 轮：删掉「原始图上有明显长红条」的帧
- 顶部视图切到 **「原始」**，并勾上 **「残差」**。
- 在原始图上，每个角点会画出**残差线**；拟合差的角点会拖出**明显的长红条**。
- 哪张帧上出现**明显长红条** → 选中它 → 「⌧ 删除所选」删掉。

### 再次运行标定
- 删完后再点 **「运行标定」**。
- 看 rms：**进到 0.25 px 以内（绿灯）即可**。
- 若还没到：重复两轮筛图，或按第 2 步补拍更多**倾斜 / 边缘**帧再标。
- 若状态条出现「dropped N ill-conditioned」之类：那是太正对/角点铺不开被自动丢的帧，补拍倾斜帧。

> rms 红绿灯阈值：绿 **< 0.25 px**、黄 **< 0.5 px**、红 ≥ 0.5。目标是**绿灯 < 0.25**。

---

## 5. 验证去畸变

顶部视图切 **「分屏」**（左原始 / 右校正后）或 **「校正后」**：
- 看门框、桌沿、墙角这种本该是直线的东西，校正后**变直了** = 成功；重点看**画面边缘**还弯不弯。
- 「去畸变预览」里的 **balance**（默认 0.6）/ **fov 缩放** 只影响**预览显示**，不改标定结果，随便调着看。
- **「对比方法」** 视图可对比 remap 与 undistortImage 两种去畸变方式。

---

## 6. 保存

右下角 **「↑ 保存」**，存成 `fisheye.yaml`（或 `.json`）。内容是 **K** 与畸变 **D = [k₁,k₂,k₃,k₄]**。
以后 **「↓ 加载」** 可载回查看 / 继续用。

## 快捷键（已验证可用）

| 键 | 作用 |
|---|---|
| **空格** | 抓拍当前帧（添加） |
| **← / →** | 翻看 / 选中已存帧 |

## 流程总结（照这个顺序）

1. 源 → `ros2` → 话题 `image_rect_compressed`；标定靶标 → 棋盘格、`11 × 8`、方格 `0.045`；投影模型默认 等距 (equidistant)。
2. 数据集 → 📁 选择文件夹（空文件夹）。
3. 手举板按第 2 步清单拍约 30 张（覆盖边缘 + 各种倾斜 ±30~45°，把 8×5 覆盖网格铺满）。
4. 点「运行标定」（第一次）。
5. 第 1 轮：帧条里**数值发黄**的 → 删除所选。
6. 第 2 轮：「原始」+「残差」下**有明显长红条**的 → 删除所选。
7. 再点「运行标定」 → **rms 进 0.25 px 以内即可**。
8. 「分屏」视图验证去畸变 → 「↑ 保存」成 `fisheye.yaml`。
